datafort

Rache Bartmoss' Datafort was created as an unofficial ressource for R. Talsorian's tabletop roleplaying game Cyberpunk 2020, first published in 1990.

As most of the communities does, this website focuses on the 1993 v2.01 edition.

This website features three sections :
- TOOLS, where you'll find tools for GMs and players
- DATA, where you'll find lists of most cyberpunk things (corporations, cyberware, vehicles, weapons) and their corresponding references in the official books
- FILES, where you'll find necessary books to get started as well as homebrews I have designed

This website was built for my friends on the scraps of what fans had previously gathered and evolved. For the sake of the tools, some homebrew elements appear in most databases (storages containers for instance).
-> For a serious, quality Cyberpunk 2020 webtool, please consult Cybersmily's website

NEW — Multiplayer (beta) : the GM runs a small free local app, players sync their sheets live on the same Wi-Fi. No cloud, no account. Joining a campaign? →

PLAY TOGETHER — LOCAL, FREE, NO CLOUD

Multiplayer is local-first: the GM runs a small free app on their computer (the GM Hub). Everyone on the same Wi-Fi connects to it. No account, no cloud, no server fees. Player sheets sync live to the GM's screen, and the GM can edit any sheet in real time.

If you're the GM

  1. Download & open the GM Hub (below).
  2. It opens your dashboard and shows links to share.
  3. Send each player their sheet link.

If you're a player

  1. Nothing to install.
  2. Open the link your GM sends, on the same Wi-Fi.
  3. Edit your sheet — it syncs to the GM.

Download the GM Hub

macOS .dmg Windows .exe

Not yet code-signed: macOS → right-click ▸ Open · Windows → More info ▸ Run anyway. On first launch, allow it through the firewall so players can connect.

How a session works

  1. The GM opens the Hub → a folder of editable sheet files (.json) is created.
  2. The GM shares a player link like http://192.168.1.42:8787/cs.html?campaign=main&sheet=Player_1.
  3. Players open it; the page is served by the GM's machine, so live sync just works (plain http:// on the LAN — no padlock, that's normal).
  4. Offline edits merge automatically once reconnected.
